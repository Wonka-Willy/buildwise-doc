Build a personal library of effective prompts that you can reuse and customize with variables. 
Keep the prompts that work well and reuse them across conversations and projects; save time and maintain consistency across your AI interactions.

<Info>
The Prompt Library helps you capture successful prompts and turn them into reusable templates for common tasks.
</Info>

### Creating Your First Saved Prompt

<Steps>
<Step title="Identify Successful Prompts">
When you get great results from a prompt, mark it as a candidate for your library. Look for prompts that:
- Produce consistently good outputs
- Solve common recurring tasks
- Include useful structure or formatting
- Minimize errors through thorough testing

<Tip>
Consider using the bookmark feature to store prompts in your personal library before officially adding them to your prompt library.
</Tip>

</Step>


<Step title="Save to Your Library">
To save a prompt to your library, navigate to the right panel and select the "Prompts" section. From there, create or acces any prompt in your library.

<Frame caption="Locate the Library Icon">
<img src="/images/prompt-library.png" alt="Library icon in the toolbar" />
</Frame>


</Step>

<Step title="Fill in Prompt Details">
In the prompt creation form, provide:
- **Prompt name** (required): A descriptive name for your prompt
- **Text** (required): The full prompt text with any [_variables_](#variable-management) you need
- **Description** (optional): A short description displayed when using the prompt
- **Command name** (optional): Create a custom command shortcut to invoke this prompt quickly

<Tip>
Give your saved prompts descriptive names that explain their purpose, like "Code Review Template" or "Meeting Notes Summary."
</Tip>

</Step>

<Step title="Save and Use">
Once saved, your prompt appears in the Prompts library where you can:
- Search for prompts by name using the filter
- Click on a prompt to use it in a conversation
- Fill in any variables when using the prompt

<Check>
Quick usage: Prompts can be invoked by typing `/` followed by their name or your custom command directly from any conversation.
</Check>

<Frame caption="Prompt example with a custom command '/exp'">
<img src="/images/slash-exp.png" alt="Example of using a saved prompt in a conversation" />
</Frame>


</Step>
</Steps>

## Variable Management

Make your prompts flexible and reusable with variables that you customize each time you use a template. WonkaChat supports three types of variables: **basic variables** for custom input, **special variables** that auto-populate with system information, and **dropdown variables** for predefined options.

<Tabs>
<Tab title="Basic Variable Syntax">
Use double curly braces to define variables in your prompts:

```
Analyze the {{component_name}} component and suggest improvements 
for {{specific_aspect}}. Focus on {{priority_area}} and provide 
code examples.
```

When using this prompt, you'll be prompted to fill in:
- `{{component_name}}` → "UserAuthentication"
- `{{specific_aspect}}` → "performance"
- `{{priority_area}}` → "database queries"
</Tab>

<Tab title="Special Variables">
WonkaChat provides built-in special variables that auto-populate:

- `{{current_date}}` - Current date (e.g., "October 7, 2025")
- `{{current_datetime}}` - Current local date and time
- `{{utc_iso_datetime}}` - Current UTC time in ISO format
- `{{current_user}}` - Your username

Example:
```
Create a report for {{current_user}} dated {{current_date}}.
```
</Tab>

<Tab title="Dropdown Variables">
Create dropdown menus with predefined options:

```
{{variable_name:option1|option2|option3}}
```

Example:
```
Summarize this document as {{format:bullet points|paragraphs|table}}.
```

When using the prompt, you'll see a dropdown menu with the specified options.
</Tab>
</Tabs>

### Common Variable Types

<AccordionGroup>
<Accordion title="Content Variables">
Replace specific content like file names, code snippets, or data:

```
Debug the following {{programming_language}} code and fix {{error_type}}:

{{code_snippet}}

Provide explanations for each fix.
```
</Accordion>

<Accordion title="Format Variables with Dropdowns">
Control output format or structure using dropdown variables:

```
Summarize this document as {{format_type:bullet points|paragraphs|table|timeline}}.

{{document_content}}
```
</Accordion>

<Accordion title="Scope Variables">
Define the breadth or depth of the response:

```
Explain {{concept}} at a {{complexity_level:beginner|intermediate|expert}} level, including {{number}} examples.
```
</Accordion>

<Accordion title="Context Variables">
Provide situational context:

```
As a {{role}}, review this {{artifact_type}} for {{project_name}}. 
Consider {{specific_requirements}} and provide feedback on 
{{focus_areas}}.
```
</Accordion>
</AccordionGroup>

<Check>
Well-designed variables make your prompts adaptable to many situations without losing effectiveness.
</Check>

## Prompt Library Examples

Ready-to-use prompt templates for common scenarios.

### Development and Coding

<AccordionGroup>
<Accordion title="Code Review Template" icon="code">
**Purpose**: Get structured code reviews with specific feedback

**Prompt**:
```
Review the following {{language}} code for {{component_name}}.

{{code_block}}

Please evaluate:
1. Code quality and readability
2. Performance considerations
3. Security vulnerabilities
4. Best practices compliance
5. {{additional_focus}}

Provide specific suggestions with code examples where applicable.
```

**Variables**:
- `{{language}}`: Programming language (e.g., JavaScript, Python)
- `{{component_name}}`: Name or purpose of the code
- `{{code_block}}`: The actual code to review
- `{{additional_focus}}`: Specific areas of concern (optional)
</Accordion>

<Accordion title="Bug Diagnosis Assistant" icon="bug">
**Purpose**: Systematically diagnose and fix bugs

**Prompt**:
```
I'm experiencing {{bug_description}} in {{system_component}}.

Environment:
- Language/Framework: {{tech_stack}}
- Version: {{version}}
- Environment: {{environment:development|staging|production}}

Error message:
{{error_message}}

Relevant code:
{{code_snippet}}

Please:
1. Identify the likely root cause
2. Explain why this error occurs
3. Provide a fix with code examples
4. Suggest how to prevent similar issues
```

**Variables**:
- `{{bug_description}}`: Brief description of the bug
- `{{system_component}}`: Where the bug occurs
- `{{tech_stack}}`: Technologies involved
- `{{version}}`: Version numbers
- `{{environment}}`: Development/staging/production (dropdown)
- `{{error_message}}`: Full error output
- `{{code_snippet}}`: Relevant code context
</Accordion>

<Accordion title="Documentation Generator" icon="file-lines">
**Purpose**: Create comprehensive documentation

**Prompt**:
```
Generate {{doc_type:API docs|README|inline comments}} documentation for the following {{language}} code:

{{code_block}}

Include:
- Purpose and functionality overview
- Parameter descriptions with types
- Return value details
- Usage examples
- {{additional_sections}}

Format the output as {{output_format:Markdown|JSDoc|reStructuredText}}.
```

**Variables**:
- `{{doc_type}}`: Type of documentation (dropdown)
- `{{language}}`: Programming language
- `{{code_block}}`: Code to document
- `{{additional_sections}}`: Edge cases, error handling, etc.
- `{{output_format}}`: Format for the output (dropdown)
</Accordion>
</AccordionGroup>

### Data Analysis and Research

<AccordionGroup>
<Accordion title="Data Analysis Framework" icon="chart-column">
**Purpose**: Analyze data with structured insights

**Prompt**:
```
Analyze the following {{data_type:sales|user behavior|performance metrics}} data for {{business_context}}:

{{data_content}}

Please provide:
1. Key findings and trends
2. Statistical summary
3. Anomalies or outliers
4. {{specific_metrics}} analysis
5. Actionable recommendations

Focus particularly on {{priority_area}}.
```

**Variables**:
- `{{data_type}}`: Type of data (dropdown)
- `{{business_context}}`: What business question to answer
- `{{data_content}}`: The actual data
- `{{specific_metrics}}`: KPIs or metrics to calculate
- `{{priority_area}}`: Main area of interest
</Accordion>

<Accordion title="Research Summarizer" icon="magnifying-glass">
**Purpose**: Synthesize research materials

**Prompt**:
```
Summarize this {{content_type:article|paper|report|documentation}} about {{topic}}:

{{content}}

Provide:
1. Executive summary (2-3 sentences)
2. Main points (bullet list)
3. Key takeaways for {{target_audience}}
4. Relevant quotes or data
5. Connections to {{related_topic}} (if applicable)

Format: {{output_format:bullets|paragraphs|table}}
Length: {{word_limit}} words maximum
```

**Variables**:
- `{{content_type}}`: Type of content (dropdown)
- `{{topic}}`: Subject matter
- `{{content}}`: The text to summarize
- `{{target_audience}}`: Who will read the summary
- `{{related_topic}}`: Context to consider
- `{{output_format}}`: Output format (dropdown)
- `{{word_limit}}`: Maximum length
</Accordion>
</AccordionGroup>

### Writing and Communication

<AccordionGroup>
<Accordion title="Email Composer" icon="envelope">
**Purpose**: Draft professional emails

**Prompt**:
```
Write a {{tone:formal|casual|apologetic|celebratory}} email to {{recipient}} about {{subject}}.

Context:
{{background_info}}

Key points to cover:
{{main_points}}
```

## Tips for a clean prompt library

<AccordionGroup>
<Accordion title="Use Descriptive Variable Names">
Choose variable names that clearly indicate what information is needed. Use `{{user_feedback}}` instead of `{{text}}` or `{{input}}`.
</Accordion>

<Accordion title="Leverage Special Variables">
Take advantage of `{{current_date}}`, `{{current_user}}`, and other built-in variables to automatically include context without manual input.
</Accordion>

<Accordion title="Create Dropdown Options for Common Choices">
Use the dropdown syntax `{{variable:option1|option2|option3}}` for variables with a limited set of valid values to reduce typing and errors.
</Accordion>

<Accordion title="Keep Prompts Focused">
Create separate prompts for different tasks rather than one overly complex prompt. 
Focused prompts are not only easier to maintain and reuse, but also yield better results.
</Accordion>
</AccordionGroup>
